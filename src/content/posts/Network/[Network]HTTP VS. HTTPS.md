# HTTP vs HTTPS

이 문서는 HTTP와 HTTPS의 동작 원리, OSI 7계층에서의 위치, 그리고 두 프로토콜의 공통점과 차이점을 설명합니다.

## OSI 7계층 모델

OSI 7계층 모델은 네트워크 통신을 계층별로 나누어 설명하는 모델입니다. 각 계층은 특정한 네트워크 기능을 담당하며, 
상위 계층은 하위 계층의 기능을 바탕으로 동작합니다. HTTP와 HTTPS는 OSI 모델에서 `7계층(애플리케이션 계층)` 에 속합니다.

## HTTP 동작 원리와 OSI 7계층에서의 역할

HTTP(HyperText Transfer Protocol)는 **웹 서버와 클라이언트 간에 데이터를 주고받는 프로토콜**입니다. HTTP는 OSI 7계층의 애플리케이션 계층에서 동작하며, 하위 계층의 기능을 이용해 데이터를 전송합니다.

### HTTP 동작 과정

1. **애플리케이션 계층 (7계층)**:
    - 사용자(Client)가 웹 브라우저에서 URL을 입력하면, 브라우저는 HTTP 요청 메시지를 생성합니다. 이 요청에는 메서드(GET, POST 등), 헤더, URL, 기타 데이터가 포함됩니다.

2. **표현 계층 (6계층)**:
    - 요청 메시지의 인코딩 및 데이터 형식이 정의됩니다. HTTP 메시지는 텍스트로 표현되며, 인코딩된 데이터가 필요할 경우 여기서 처리됩니다.

3. **세션 계층 (5계층)**:
    - 브라우저와 서버 간의 세션이 관리됩니다. HTTP 자체는 상태를 유지하지 않지만, 세션 쿠키나 세션 ID를 통해 연결 상태를 추적할 수 있습니다.

4. **전송 계층 (4계층)**:
    - TCP 프로토콜을 사용하여 신뢰성 있는 데이터 전송이 이루어집니다. HTTP 요청 메시지는 TCP 세그먼트로 나누어집니다.

5. **네트워크 계층 (3계층)**:
    - IP 프로토콜을 사용하여 데이터가 클라이언트와 서버 간에 라우팅됩니다. IP 주소를 통해 목적지 서버에 데이터를 전달합니다.

6. **데이터 링크 계층 (2계층)**:
    - MAC 주소를 사용하여 데이터가 동일 네트워크 내의 장치 간에 전송됩니다. 이 계층에서 데이터 프레임이 형성됩니다.

7. **물리 계층 (1계층)**:
    - 전기적 신호로 변환된 데이터가 물리적 매체(예: 이더넷 케이블)를 통해 전송됩니다.

8. **서버 응답**:
    - 서버는 클라이언트의 요청을 처리하고, HTTP 응답 메시지를 동일한 OSI 계층을 거쳐 클라이언트에게 전달합니다.

## HTTPS 동작 원리와 OSI 7계층에서의 역할

HTTPS(HyperText Transfer Protocol Secure)는 **HTTP에 SSL/TLS(Transport Layer Security)를 추가하여 보안을 강화한 프로토콜**입니다. HTTPS는 OSI 7계층에서 동작하며, 특히 표현 계층과 세션 계층에서 보안 기능이 강화됩니다.

### HTTPS 동작 과정

1. **애플리케이션 계층 (7계층)**:
    - 클라이언트가 HTTPS URL을 입력하면, 브라우저는 HTTPS 요청 메시지를 생성합니다. 이 메시지 역시 HTTP 메시지와 유사합니다.

2. **표현 계층 (6계층)**:
    - SSL/TLS를 사용해 데이터가 암호화됩니다. 클라이언트와 서버는 핸드셰이크를 통해 세션 키를 생성하고, 이후의 통신은 이 대칭 키로 암호화됩니다.

3. **세션 계층 (5계층)**:
    - SSL/TLS 핸드셰이크를 통해 보안 세션이 설정됩니다. 세션 관리와 함께, 보안 설정이 이 계층에서 이루어집니다.

4. **전송 계층 (4계층)**:
    - TCP 프로토콜을 통해 암호화된 데이터가 신뢰성 있게 전송됩니다. HTTPS도 TCP 세그먼트를 사용하여 데이터를 전송합니다.

5. **네트워크 계층 (3계층)**:
    - IP 프로토콜을 사용해 암호화된 데이터가 목적지 서버로 라우팅됩니다.

6. **데이터 링크 계층 (2계층)**:
    - MAC 주소를 사용해 암호화된 데이터가 동일 네트워크 내의 장치 간에 전송됩니다.

7. **물리 계층 (1계층)**:
    - 전기적 신호로 변환된 암호화된 데이터가 물리적 매체를 통해 전송됩니다.

8. **서버 응답**:
    - 서버는 클라이언트의 요청을 처리하고, 암호화된 HTTPS 응답 메시지를 동일한 OSI 계층을 거쳐 클라이언트에게 전달합니다.

## HTTP와 HTTPS의 공통점

- **웹 데이터 전송**: 둘 다 클라이언트와 서버 간의 데이터 전송에 사용됩니다.
- **요청-응답 모델**: HTTP와 HTTPS 모두 요청-응답 모델을 사용하여 데이터를 주고받습니다.
- **사용하는 메서드**: GET, POST, PUT, DELETE 등 같은 HTTP 메서드를 사용합니다.
- **OSI 7계층**: 둘 다 OSI 모델의 7계층에서 동작하며, 하위 계층을 통해 데이터를 전달합니다.

## HTTP와 HTTPS의 차이점

| **특징**        | **HTTP**                 | **HTTPS**                  |
|---------------|--------------------------|----------------------------|
| **보안**        | 암호화되지 않은 평문 데이터 전송       | SSL/TLS를 사용하여 데이터 암호화      |
| **포트 번호**     | 기본적으로 포트 80 사용           | 기본적으로 포트 443 사용            |
| **속도**        | 상대적으로 빠름                 | 암호화/복호화로 인해 약간 느림          |
| **인증서 사용**    | 사용하지 않음                  | SSL/TLS 인증서를 사용하여 서버 인증    |
| **보안 공격 위험성** | 도청, 중간자 공격, 데이터 변조 등에 취약 | 데이터 도청, 중간자 공격, 변조에 강함     |
| **데이터 전송**    | 평문 데이터 전송                | 암호화된 데이터 전송                |
| **사용 사례**     | 보안이 중요하지 않은 웹 페이지        | 로그인, 금융 거래 등 보안이 중요한 웹 페이지 |

## 결론

HTTP와 HTTPS는 웹에서 데이터를 주고받기 위한 프로토콜로, HTTPS는 HTTP의 보안 문제를 해결하기 위해 SSL/TLS를 추가한 확장된 버전입니다. 두 프로토콜은 OSI 7계층의 애플리케이션 계층에서 동작하며, HTTPS는 표현 계층과 세션 계층에서 암호화를 추가하여 보다 안전한 웹 통신을 제공합니다.
